## 🚀 Accomplishments

- **Architectural Implementation**:
  Successfully built a "Vertical Slice" for the `add` command using Layered Architecture
  (App -> Middleware -> Route -> Controller -> Service -> Model -> Repository).

- **Data Integrity**:
  Created a robust `Task` Model with a constructor
  that ensures data consistency
  (ID, Description, Status, and ISO Timestamps).

- **Repository Pattern**:
  Developed reusable `read.js` and `write.js` modules using `node:fs/promises`,
  including logic to handle missing directories and empty files.

- **Advanced Validation**:
  Implemented multiple Middlewares to "Fail Fast,"
  including argument length checks and description validation.

- **Error Handling**:
  Integrated a custom `AppError` class to handle system vs.
  logic errors professionally.

- **Git Best Practices**:
  Cleaned the repository by untracking sensitive data,
  implementing `.gitignore`,
  and establishing a standard for template files (`tasks.json.example`).

## 🛠️ Technical Logic Refinement

- **ID Generation**:
  Created a pure utility to handle auto-incrementing IDs based on the current dataset.

- **Edge Case Resolution**:
  Fixed a `SyntaxError` crash caused by empty JSON files
  by returning `null` from the repository layer.

- **Standardization**:
  Adopted the `.json.example` naming convention
  for better tool compatibility and developer experience.

## 📝 Rules of the System

1. **The Brain (Service)** never speaks to the **View (CLI)**.
2. **The Hands (Repository)** are "dumb" and only handle file I/O.
3. **The Gatekeeper (Middleware)** handles format validation and terminates invalid processes early.
4. **The Conductor (Controller)** coordinates the flow and catches errors to pass them to the View. Testing
